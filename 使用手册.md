## 1. 系统概述

Gas System 是一套面向化工园区的多源废气智能治理系统原型，集成了**多源数据融合**、**VOCs浓度预测**、**可视化监控看板**三大核心功能。

### 核心功能

| 模块 | 功能 | 技术 |
|------|------|------|
| 数据融合 | JSON/CSV/Excel 多格式数据接入、清洗、对齐、存储 | Python + Flask + MySQL |
| VOCs预测 | 基于 LSTM 的未来6小时 VOCs 浓度趋势预测 | TensorFlow 2.21 + Keras |
| 预警引擎 | 超限自动预警 (80mg/m³)，三级分级 (高/中/低) | Flask + MySQL |
| 监控看板 | 实时数据、预测曲线、预警列表一体化展示 | Vue 3 + ECharts |
| 报告导出 | CSV 格式历史数据导出 | Flask Response |

### 系统架构

```
┌─────────────────────────────────────────────┐
│              表现层: Vue.js + ECharts         │
│           http://localhost (Nginx :80)        │
├─────────────────────────────────────────────┤
│            业务逻辑层: Flask RESTful API       │
│          http://localhost:5000                │
├─────────────────────────────────────────────┤
│        智能与数据层: TensorFlow + MySQL        │
│         LSTM模型 (124K参数) + MySQL :3306      │
├─────────────────────────────────────────────┤
│         部署层: Docker + docker-compose        │
└─────────────────────────────────────────────┘
```

## 3. 快速开始 (5分钟部署)

### Docker 一键部署（推荐）

```bash
# 1. 进入项目目录
cd Gas-System

# 2. 一键构建并启动所有服务
docker-compose up -d --build

# 3. 等待约 60 秒（首次启动需下载镜像、初始化 MySQL、安装依赖）

# 4. 验证服务
curl http://localhost:5000/api/realtime    # 后端API
curl http://localhost/                      # 前端看板

# 5. 浏览器打开
# http://localhost
```

### 停止服务

```bash
docker-compose down          # 停止并删除容器
docker-compose down -v       # 同时删除数据库数据卷
```

## 4. 本地开发部署

### 4.1 后端启动

```bash
# 进入后端目录
cd backend

# 创建虚拟环境
python -m venv .venv

.venv\Scripts\activate
```

## 6. API 接口说明

### 基础信息

- Base URL: `http://localhost:5000`
- 内容类型: `application/json`
- 字符编码: `UTF-8`

### 接口列表

#### 6.1 文件上传 — `POST /api/data/upload`

支持 CSV (.csv) 和 Excel (.xlsx/.xls) 文件上传。

**请求参数** (multipart/form-data):

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `file` | File | 是 | CSV 或 Excel 文件 |
| `data_type` | String | 是 | `emission` / `equipment` / `weather` |

**请求示例**:
```bash
# CSV 上传
curl -X POST http://localhost:5000/api/data/upload \
  -F "file=@emission_data.csv" \
  -F "data_type=emission"

# Excel 上传
curl -X POST http://localhost:5000/api/data/upload \
  -F "file=@weather_data.xlsx" \
  -F "data_type=weather"
```

**成功响应** (200):
```json
{
  "message": "导入成功",
  "imported_rows": 2160,
  "before_completeness": 95.2,
  "after_completeness": 100.0
}
```

**错误响应**:
```json
{"error": "无效文件，仅支持 .csv / .xlsx / .xls"}
{"error": "缺少或无效的 data_type，可选: emission, equipment, weather"}
```

---

#### 6.2 实时数据接入 — `POST /api/data/realtime`

**请求体** (JSON):

```json
{
  "data_type": "emission",
  "records": [
    {
      "sensor_id": "S01",
      "timestamp": "2026-05-28 14:00:00",
      "voc_concentration": 65.3,
      "nox_concentration": 26.1,
      "so2_concentration": 5.2
    }
  ]
}
```

**必填字段**:
- `emission` 类型: `sensor_id`, `timestamp`
- `equipment` 类型: `equipment_id`, `timestamp`
- `weather` 类型: `station_id`, `timestamp`

**请求示例**:
```bash
curl -X POST http://localhost:5000/api/data/realtime \
  -H "Content-Type: application/json" \
  -d '{"data_type":"weather","records":[{"station_id":"W01","timestamp":"2026-05-28 14:00:00","temperature":25.5,"humidity":60.2,"wind_speed":3.5,"wind_direction":180}]}'
```

**成功响应** (200):
```json
{"message": "实时数据接入成功", "processed_records": 1}
```

### 错误码说明

| HTTP 状态码 | 含义 | 常见原因 |
|-------------|------|----------|
| 200 | 成功 | - |
| 400 | 请求参数错误 | 缺少必填字段、文件格式不支持、数据不足 |
| 500 | 服务器内部错误 | 数据库连接失败、模型推理异常 |
| 503 | 服务不可用 | 模型文件缺失 (需先运行 train_model.py) |

## 7. 数据格式说明

### 7.1 排放数据 (emission)

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `sensor_id` | String | 是 | 传感器ID, 如 "S01" |
| `timestamp` | DateTime | 是 | 采样时间, 格式 "YYYY-MM-DD HH:MM:SS" |
| `voc_concentration` | Float | 否 | VOCs浓度 (mg/m³) |
| `nox_concentration` | Float | 否 | NOx浓度 (mg/m³) |
| `so2_concentration` | Float | 否 | SO₂浓度 (mg/m³) |

### 7.2 气象数据 (weather)

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `station_id` | String | 是 | 气象站ID, 如 "W01" |
| `timestamp` | DateTime | 是 | 观测时间 |
| `temperature` | Float | 否 | 温度 (°C) |
| `humidity` | Float | 否 | 相对湿度 (%) |
| `wind_speed` | Float | 否 | 风速 (m/s) |
| `wind_direction` | Float | 否 | 风向 (度, 0-360) |

### 7.3 设备数据 (equipment)

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `equipment_id` | String | 是 | 设备ID, 如 "E01" |
| `timestamp` | DateTime | 是 | 记录时间 |
| `operating_load` | Float | 否 | 运行负荷 (%) |
| `production_phase` | String | 否 | 生产时段, 如 "正常"/"低负荷" |
| `status` | String | 否 | 设备状态, 如 "running" |

### 7.4 CSV 文件模板

## 8. 模型训练指南

### 8.1 训练流程

```bash
cd backend

# 步骤1: 生成训练数据 (首次/数据更新后)
python generate_real_data.py

# 步骤2: 数据分析 + 特征工程 (生成 scaler.pkl)
python data_analysis.py

# 步骤3: 训练 LSTM 模型
python train_model.py
```

### 8.2 输出文件

| 文件 | 说明 |
|------|------|
| `correlation_heatmap.png` | VOCs与各因素相关性热力图 |
| `engineered_features.npz` | 特征矩阵 (X) + 标签向量 (y) |
| `scaler.pkl` | 标准化器 (预测时必须) |
| `feature_cols.npy` | 特征列名称列表 |
| `best_model.keras` | 验证集最优模型 |
| `vocs_lstm_model.keras` | 最终模型 (预测服务加载) |
| `model_architecture.png` | 模型结构可视化 |
| `training_history.png` | 训练/验证损失曲线 |
| `prediction_vs_actual_*.png` | 预测 vs 真实对比图 |

### 8.3 模型性能

| 指标 | 数值 | 要求 | 状态 |
|------|------|------|------|
| 参数量 | 124,902 | <500,000 | OK |
| 输入 | (24时间步, 14特征) | 多变量时序 | OK |
| 输出 | 6个未来值 | 未来6小时 | OK |
| 准确率 (1-MAPE) | 82.61% | ≥75% | OK |
| 推理耗时 | ~99ms | ≤2s | OK |

### 8.4 新增真实数据后重新训练

```bash
# 1. 将新的 CSV/Excel 数据上传到数据库

  -F "file=@new_data.csv" -F "data_type=emission"

# 3. 重启后端服务加载新模型
```

## 10. 常见问题排查

### 10.1 后端连接不上数据库

**现象**: `RuntimeError: 'cryptography' package is required`

**解决**:
```bash
pip install cryptography
```

**现象**: 接口返回 500, 日志显示 MySQL 连接拒绝

**解决**:
1. 确认 MySQL 已启动: `docker ps | grep mysql`
2. 确认数据库已创建: `mysql -uroot -p123456 -e "SHOW DATABASES;"`
3. 确认 `app.py` 中数据库连接参数正确

### 10.2 预测接口返回数据不足

**现象**: `{"error": true, "message": "有效数据不足（当前 X 条，需要至少 30 条）"}`

**解决**: 向数据库插入至少 30 条以上跨小时的历史数据。
```bash
python generate_real_data.py  # 自动生成 2160 条并导入
```

### 10.3 预测准确率低

**排查步骤**:
1. 检查数据量是否充足 (建议 ≥2000 条)
2. 运行 `python data_analysis.py` 查看特征相关性
3. 调整模型参数 (增加 LSTM 单元数、增长回看窗口)
4. 增加更多训练数据

### 10.4 前端看板数据不刷新

**排查步骤**:
1. 检查浏览器控制台 Network 面板 API 请求是否正常
2. 确认后端服务可访问: `curl http://localhost:5000/api/realtime`
3. 检查前端代理配置 (开发模式下 `vue.config.js` 中的 proxy 设置)

### 10.5 Docker 容器启动失败

**现象**: 容器状态 `Restarting` 或 `Exited`

**解决**:
```bash
# 查看容器日志
docker logs gas_backend
docker logs gas_mysql

# 清除旧容器和镜像重建
docker-compose down -v
docker-compose up -d --build
```

### 10.6 端口被占用

**现象**: `port is already allocated`

**本地开发**: 修改 `app.py` 中的 `port=5000` 或前端 `vue.config.js` 中的端口。

**Docker**: 修改 `docker-compose.yml` 中的端口映射, 如 `"5001:5000"`。
