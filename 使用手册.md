# Gas System — 面向化工园区的多源废气智能治理系统

## 零基础使用手册

> **这是一份写给完全零基础用户的操作指南。** 你不需要懂编程，只要会复制粘贴命令、会用浏览器，就能在 10 分钟内把这个系统跑起来。

---

## 新手必读

### 这个系统是做什么的？

一句话：**帮化工厂的环保管理人员预测废气会不会超标。**

它会自动做三件事：
1. **收集数据** — 把废气浓度、天气、设备运转情况自动汇总到一起
2. **预测未来** — 告诉你接下来 6 小时废气浓度会不会超标
3. **弹窗报警** — 如果真的快超标了，马上弹警告提醒你

### 运行起来长什么样？

最终你会看到这样的一个网页：

```
┌──────────────────────────────────────────────────┐
│  Gas System 监控看板              📄导出报告 ●运行中│
├──────────────┬───────────────────────────────────┤
│  VOCs 浓度   │                                   │
│  64.4 mg/m³  │    📈 预测趋势图                    │
│              │   蓝色线=过去数据  黄色线=未来预测    │
│  温度 29.9°C │                                   │
│  湿度 47.0%  ├───────────────────────────────────│
│  风速 2.7m/s │    ⚠ 预警列表 (超标时才会出现)       │
│              │   时间    │指标    │级别   │详情     │
└──────────────┴───────────────────────────────────┘
```

### 你需要准备什么？

只需要一台 Windows 电脑，其他什么都不需要。下面会一步步教你。

## 方式一：Docker 一键部署（推荐，最简单）

> **适用人群**：完全零基础，想最快看到效果的用户。
>
> **前提**：电脑需要安装 Docker Desktop（下面会教你怎么装）。
>
> **耗时**：首次约 15 分钟（主要是下载时间），之后每次只需 30 秒。

### 第 1 步：安装 Docker Desktop

Docker 就像一个盒子，把整个系统打包进去，你不需要单独装 Python、MySQL 等任何东西。

1. 打开浏览器，访问: **https://www.docker.com/products/docker-desktop/**
2. 点击页面中间的蓝色大按钮 **"Download Docker Desktop"**
3. 下载完成后，双击安装文件（文件名类似 `Docker Desktop Installer.exe`）
4. 一路点 **"下一步"** → **"安装"** → **"完成"**，全部用默认选项即可
5. 安装完成后，桌面上会出现一个鲸鱼图标 🐳，双击打开它
6. 第一次打开会要求你注册账号，点 **"Sign up"** 用邮箱注册一个免费账号
7. 注册完登录，等鲸鱼图标右下角变成绿色，就说明 Docker 启动好了

> ✅ **验证**：打开"命令提示符"（按键盘 `Win+R`，输入 `cmd` 回车），在黑色窗口里输入：
> ```
> docker --version
> ```
> 如果显示类似 `Docker version 29.x.x`，说明安装成功。

### 第 2 步：打开命令窗口

1. 按键盘 `Win + R`，输入 `cmd`，点确定
2. 一个黑色窗口会打开 — 这是"命令提示符"，你只需要在里面**复制粘贴**命令即可

### 第 3 步：进入项目目录

把下面这行**复制**（`Ctrl+C`），然后在黑色窗口里**粘贴**（`Ctrl+V`），按回车：

```cmd
cd /d "e:\文件\大二下作业\Gas System"
```

### 第 4 步：一键启动

复制下面这条命令，粘贴到黑色窗口，按回车：

```cmd
docker-compose up -d --build
```

此时屏幕上会出现很多文字在滚动 — **这是正常的**，它在自动下载和安装各种东西。

> ⏳ **首次启动需要等待 5-15 分钟**（取决于网速），因为它要下载约 500MB 的文件。
>
> 你会看到类似这样的文字：
> ```
> [+] Building 150.2s
> [+] Running 3/3
>  Container gas_mysql    Started
>  Container gas_backend  Started
>  Container gas_frontend Started
> ```
>
> 看到 "Started" 就说明启动成功了！

### 第 5 步：确认系统在运行

在黑色窗口输入：

```cmd
docker ps
```

你应该看到类似这样的表格：

```
CONTAINER ID   IMAGE                STATUS                   PORTS
xxxxxxxxxx     gassystem-frontend   Up xx minutes            0.0.0.0:80->80/tcp
xxxxxxxxxx     gassystem-backend    Up xx minutes            0.0.0.0:5000->5000/tcp
xxxxxxxxxx     mysql:8.0            Up xx minutes (healthy)   0.0.0.0:3307->3306/tcp
```

> ✅ 只要看到三个都在 "Up" 状态，就说明一切正常！

### 第 6 步：打开浏览器看效果

1. 打开你电脑上的浏览器（Chrome、Edge、Firefox 随便哪个）
2. 在地址栏输入：**`http://localhost`**
3. 按回车

你应该看到一个深色背景的监控大屏，左边有 4 个数据卡片（VOCs浓度、温度、湿度、风速），右边有预测曲线图。

> 🎉 **看到这个页面，说明部署成功！**

### 以后怎么用？

- **启动系统**：打开 Docker Desktop（鲸鱼图标变绿），然后重复第 2-4 步即可
- **停止系统**：在黑色窗口输入 `docker-compose down`
- **完全不需要的时候**：打开 Docker Desktop，点右上角设置 → Quit Docker Desktop

## 方式二：不用 Docker，手动启动（备选）

> **适用人群**：电脑上已经有 Python 的用户。
>
> **前提**：电脑上已安装 Python 3.10+ 和 MySQL 8.0。
>
> **耗时**：首次约 20 分钟（主要是安装 Python 包）。

### 前置准备：安装 Python

如果你还没有安装 Python：

1. 打开浏览器，访问: **https://www.python.org/downloads/**
2. 页面会自动识别你的系统，点击黄色大按钮 **"Download Python"**
3. 下载完成后双击安装
4. ⚠️ **重要！** 安装界面的最下面有个勾选框 **"Add Python to PATH"**，一定要勾上！
5. 然后点 "Install Now"，等待安装完成

> ✅ **验证**：打开"命令提示符"（`Win+R` → `cmd`），输入：
> ```
> python --version
> ```
> 显示 `Python 3.x.x` 就说明安装成功。

### 前置准备：安装并配置 MySQL

如果你还没有 MySQL：

1. 打开浏览器，访问: **https://dev.mysql.com/downloads/installer/**
2. 下载 `mysql-installer-community` (约 400MB)
3. 双击安装，选择 "Developer Default"，一路下一步
4. 在设置 root 密码时，设置为：**`123456`**
5. 安装完成后，打开 MySQL Workbench 或命令行，执行：
   ```sql
   CREATE DATABASE IF NOT EXISTS gas_system CHARACTER SET utf8mb4;
   ```

### 第 1 步：打开命令窗口

按 `Win+R`，输入 `cmd`，回车。

### 第 2 步：进入后端目录

```cmd
cd /d "e:\文件\大二下作业\Gas System\backend"
```

### 第 3 步：安装 Python 依赖包

复制粘贴下面这行，按回车：

```cmd
pip install -r requirements.txt
```

> ⏳ 这步会比较慢（约 5-10 分钟），因为要下载 TensorFlow（约 350MB）。
> 如果下载失败，试试用国内镜像：
> ```
> pip install -r requirements.txt -i https://pypi.tuna.tsinghua.edu.cn/simple
> ```

### 第 4 步：生成数据并训练模型

依次执行下面三条命令（一条一条来，等上一条跑完再跑下一条）：

```cmd
python generate_real_data.py
```

> 屏幕上会显示 "Data import complete!"，说明数据生成成功。

```cmd
python data_analysis.py
```

> 屏幕上会显示 "特征矩阵 X 形状: (2125, 24, 14)" 等信息。同时会生成一张热力图 `correlation_heatmap.png`。

```cmd
python train_model.py
```

> ⏳ 约 1-2 分钟。屏幕上会显示训练进度条，最后显示 "准确率 82.61%" 等评估指标。

### 第 5 步：启动后端

```cmd
python app.py
```

此时命令窗口会显示：

```
 * Running on http://127.0.0.1:5000
```

> ⚠️ **这个窗口不要关！** 关了后端就停了。可以最小化到后台。

### 第 6 步：新开一个命令窗口，启动前端

再按 `Win+R`，输入 `cmd`，回车，打开第二个黑色窗口。

```cmd
cd /d "e:\文件\大二下作业\Gas System\frontend"
```

安装前端依赖（只需做一次）：

```cmd
npm install --registry=https://registry.npmmirror.com
```

启动前端：

```cmd
npm run serve
```

> ⏳ 约 30 秒。看到 "Compiled successfully" 就说明前端启动好了。

### 第 7 步：打开浏览器

在地址栏输入：**`http://localhost:8080`**

> 🎉 看到监控大屏说明成功！

## 使用系统：浏览器操作指南

### 看板页面说明

打开 `http://localhost` (Docker 方式) 或 `http://localhost:8080` (手动方式) 后：

| 区域 | 位置 | 内容 | 怎么用 |
|------|------|------|--------|
| 顶部标题栏 | 最上面 | 系统名称、时间、导出按钮 | 点"📄 导出报告"下载数据 |
| 左侧卡片 | 左边 | VOCs浓度、温度、湿度、风速 | **自动刷新**，无需操作 |
| 预测图表 | 右上 | 蓝色历史线 + 黄色预测线 | 鼠标悬停看具体数值 |
| 预警列表 | 右下 | 超标警告信息 | **超标时自动出现**，点某一行看详情 |

### 各功能详解

#### 📊 实时数据卡片（左侧四个框）

- 自动每 5 秒刷新一次
- 显示最新的监测数值
- 如果数据一直是 0，说明数据库里还没有数据（参考"导入你自己的数据"章节）

#### 📈 预测趋势图（右上图表）

- **蓝色实线** = 过去 24 小时实际测到的浓度
- **黄色虚线** = AI 预测的未来 6 小时浓度
- **红色虚线** = 80 mg/m³ 的预警线（超过这条线就会报警）

鼠标放到线上会弹出提示框，显示具体时间和浓度值。

#### ⚠️ 预警列表（右下表格）

| 预警级别 | 颜色 | 含义 | 触发条件 |
|----------|------|------|----------|
| 高 | 🔴 红色 | 严重，需立即处理 | 超过 3 个预测值都超标 |
| 中 | 🟡 黄色 | 注意，需关注 | 2-3 个预测值超标 |
| 低 | 🔵 蓝色 | 提醒，可关注 | 1 个预测值超标 |

> ✅ 没有超标时，这个区域显示 "暂无预警信息"

#### 📄 导出报告

点击右上角的 "📄 导出报告" 按钮，浏览器会自动下载一个 CSV 文件，用 Excel 就能打开查看历史排放数据。

## 导入你自己的数据

系统已经内置了 2160 条模拟数据。如果你想导入真实数据，有三种方式：

### 方式 A：上传 CSV 文件（最简单）

> 适合：你有一个 Excel/CSV 文件，里面记录了废气数据

1. 把你的 CSV 文件准备好（格式参考下面的模板）
2. 打开命令窗口，执行：

```cmd
curl -X POST http://localhost:5000/api/data/upload -F "file=@你的文件路径.csv" -F "data_type=emission"
```

> 把 "你的文件路径.csv" 替换成你实际的文件路径。比如文件在桌面上：
> ```
> curl -X POST http://localhost:5000/api/data/upload -F "file=@C:\Users\你的用户名\Desktop\data.csv" -F "data_type=emission"
> ```

`data_type` 参数有三个选项：
- `emission` — 废气排放数据
- `weather` — 气象数据
- `equipment` — 设备运行数据

### 方式 B：上传 Excel 文件

和 CSV 一样，把你的 `.xlsx` 文件路径替换上去即可：

```cmd
curl -X POST http://localhost:5000/api/data/upload -F "file=@C:\Users\你的用户名\Desktop\data.xlsx" -F "data_type=weather"
```

### 方式 C：用 API 实时发送数据

> 适合：有自动监测设备，想实时推送数据

```cmd
curl -X POST http://localhost:5000/api/data/realtime -H "Content-Type: application/json" -d "{\"data_type\":\"emission\",\"records\":[{\"sensor_id\":\"S01\",\"timestamp\":\"2026-06-03 10:00:00\",\"voc_concentration\":65.3}]}"
```

### CSV 文件模板

你的 CSV 文件必须包含以下列（表头名称要一模一样，大小写也要一致）：

**废气排放数据 (emission)**：
```
sensor_id,timestamp,voc_concentration,nox_concentration,so2_concentration
S01,2026-06-03 10:00:00,120.5,45.6,10.2
S01,2026-06-03 11:00:00,130.0,50.1,9.8
```

**气象数据 (weather)**：
```
station_id,timestamp,temperature,humidity,wind_speed,wind_direction
W01,2026-06-03 10:00:00,25.5,60.2,3.5,180
```

**设备数据 (equipment)**：
```
equipment_id,timestamp,operating_load,production_phase,status
E01,2026-06-03 10:00:00,72.5,正常,running
```

> 💡 **提示**：用 Excel 编辑好后，选择"文件 → 另存为 → CSV (逗号分隔)"即可。

### 导入数据后需要重新训练吗？

- **只导入了少量数据**（几十条）：不需要，直接刷新浏览器页面就能看到新数据
- **导入了大量新数据**（几百条以上）：建议重新训练模型以提高准确率。在 backend 目录下依次执行：
  ```cmd
  python data_analysis.py
  python train_model.py
  ```
  如果用的是 Docker 方式，需要重新构建：
  ```cmd
  docker-compose up -d --build backend
  ```

## 遇到问题怎么办？

### 问题 1：浏览器打开显示"无法访问此网站"

**可能原因**：服务没有启动。

**解决步骤**：
1. 打开命令窗口，输入 `docker ps`
2. 如果看到空的表格或只有 1-2 行，说明服务没启动全
3. 重新执行：`cd /d "e:\文件\大二下作业\Gas System"` → `docker-compose up -d`
4. 等 30 秒后刷新浏览器

### 问题 2：Docker 启动报错 "port is already allocated"

**含义**：端口被其他程序占用了。

**解决步骤**：
1. 用记事本打开 `e:\文件\大二下作业\Gas System\docker-compose.yml`
2. 找到 `"5000:5000"` 这一行，改成 `"5001:5000"`
3. 找到 `"80:80"` 这一行，改成 `"8080:80"`
4. 保存文件
5. 重新执行 `docker-compose up -d`
6. 浏览器访问 `http://localhost:8080` 即可

### 问题 3：页面上数据全部显示为 0

**含义**：数据库里还没有数据。

**解决步骤**（Docker 方式）：
```cmd
cd /d "e:\文件\大二下作业\Gas System\backend"
python generate_real_data.py
```
如果用的是 Docker 方式而且本地没有 Python，用下面这个命令：
```cmd
docker exec -it gas_backend python generate_real_data.py
```

### 问题 4：pip install 下载失败或被墙

**错误信息**：`Connection timeout` 或 `Could not find a version`

**解决**：使用国内镜像源：
```cmd
pip install -r requirements.txt -i https://pypi.tuna.tsinghua.edu.cn/simple
```

### 问题 5：提示 "No module named 'xxx'"

**含义**：缺少某个 Python 包。

**解决**：
```cmd
pip install xxx
```
把 `xxx` 换成报错里提到的包名即可。

### 问题 6：docker-compose 命令不存在

**含义**：Docker Desktop 没有正确安装或没有启动。

**解决**：
1. 确认桌面右下角有 Docker 鲸鱼图标
2. 鲸鱼图标不是绿色 → 右键点它 → "Start" 或 "Restart"
3. 等图标变绿后再试

## 附录：名词解释

| 名词 | 大白话解释 |
|------|-----------|
| **Docker** | 一个"盒子"，把整个系统打包进去，让你不用单独装各种东西 |
| **docker-compose** | Docker 的遥控器，一条命令就能同时启动多个盒子 |
| **容器 (Container)** | Docker 盒子里的一个小隔间，每个隔间干一件事 |
| **Flask** | 一个 Python 写的网站后端框架，负责处理数据 |
| **API** | 网站后台提供的数据接口，前端通过它拿数据 |
| **Vue.js** | 一个前端框架，负责页面的样子和交互 |
| **ECharts** | 一个画图表的工具，页面上那个好看的曲线图就是它画的 |
| **LSTM** | 一种 AI 算法，专门用来根据历史数据预测未来趋势 |
| **TensorFlow** | Google 开发的 AI 工具箱，LSTM 就是在它上面跑的 |
| **VOCs** | 挥发性有机化合物，化工厂排放的主要废气污染物 |
| **mg/m³** | 毫克每立方米，衡量空气中污染物浓度的单位 |
| **端口 (Port)** | 电脑上的"门牌号"，不同的服务用不同的门牌号 |
| **CSV** | 一种最简单的表格文件格式，用逗号分隔数据，Excel 能直接打开 |
| **JSON** | 一种数据格式，人和电脑都容易读懂 |
| **curl** | 一个命令行工具，用来在黑色窗口里测试网站接口 |
