# 面向化工园区的多源废气智能治理系统 — 项目详细方案

## 一、需求分析

### 1.1 问题定义

化工园区废气治理普遍面临三大核心问题：

**数据孤岛 — 数据分散割裂**
- 监测点多（超 50 个/园区），数据格式多样（超 10 种：JSON/CSV/Excel/XML 等）
- 各设备厂商系统独立，数据无法协同分析形成统一视图

**过程黑箱 — 预警滞后**
- 缺乏对废气浓度（尤其是受气象、工况影响的 VOCs）的有效预测手段
- 依赖人工经验判断，无法提前规避超标风险，响应被动

**调控粗放 — 决策效率低**
- 监管工具分散于多个平台，缺乏一体化监控能力
- 决策链条长，无法实时掌握园区废气治理全貌

### 1.2 用户期望

面向化工园区环保管理团队，开发可本地服务器部署的系统原型：
1. 多源数据接入与统一管理（≥3 类核心数据）
2. VOCs 浓度未来 6 小时趋势预测（准确率 ≥75%）
3. 异常时 10 秒内触发预警提示
4. 一体化 Web 看板（实时数据 + 预测曲线 + 预警列表）
5. 支持交互查询与数据报告导出
6. Docker 部署，步骤 ≤5 步

## 三、技术路线

### 3.1 数据融合模块

```
数据接入层
  ├── CSV 解析 (pandas.read_csv)
  ├── Excel 解析 (pandas.read_excel + openpyxl)
  └── JSON 解析 (Flask request.get_json)

数据清洗层 (data_cleaner.py)
  ├── 时间戳转换 (pd.to_datetime)
  ├── 缺失值填充 (前向填充 → 后向填充 → 均值填充)
  ├── 异常值处理 (负值 → 线性插值)
  └── 完整性统计 (清洗前后百分比)

时序对齐层
  ├── 分钟级 floor 对齐 (dt.floor('min'))
  ├── 三表按分钟分组取均值
  └── 小时级降采样 (用于模型训练)
```

### 3.2 VOCs 预测模块

**特征工程 (14 维)**:

| 类别 | 特征 | 数量 |
|------|------|:--:|
| 原始特征 | temperature, humidity, wind_speed, operating_load | 4 |
| 时间编码 | hour_sin, hour_cos, day_of_week_sin, day_of_week_cos, is_workday | 5 |
| 滞后特征 | voc_lag_1h, voc_lag_3h, voc_lag_6h | 3 |
| 滚动统计 | voc_rolling_6h_mean, voc_rolling_6h_std | 2 |

**模型架构**:

```
Input (24 timesteps × 14 features)
  → LSTM(128, return_sequences=True)
  → Dropout(0.3)
  → LSTM(64, return_sequences=False)
  → Dropout(0.3)
  → Dense(32, relu)
  → Dense(6)  [输出: 未来6小时VOCs浓度]
```
