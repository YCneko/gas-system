# 面向化工园区的多源废气智能治理系统 — 数据接口设计文档

> 采用 OpenAPI 3.0 规范

---

## 一、接口概览

| # | 方法 | 路径 | 功能 | 认证 |
|:--|------|------|------|:--:|
| 1 | POST | `/api/data/upload` | 文件上传 (CSV/Excel) | 无 |
| 2 | POST | `/api/data/realtime` | JSON 实时数据接入 | 无 |
| 3 | GET | `/api/realtime` | 获取最新实时数据 | 无 |
| 4 | GET | `/api/prediction` | 历史+预测数据 | 无 |
| 5 | GET | `/api/vocs/prediction` | 预测+预警一体化 | 无 |
| 6 | GET | `/api/alerts` | 预警记录列表 | 无 |
| 7 | GET | `/api/report/export` | CSV 报告导出 | 无 |

**Base URL**: `http://localhost:5000` (本地) 或 `http://localhost` (Docker Nginx 代理)

---

## 二、接口详细说明

### 2.1 文件上传

```
POST /api/data/upload
POST /api/data/upload/csv  (兼容旧接口)
```

**请求格式**: `multipart/form-data`

**参数**:

| 参数 | 类型 | 必填 | 说明 |
|------|------|:--:|------|
| file | File | 是 | 上传文件 (.csv / .xlsx / .xls) |
| data_type | String | 是 | 数据类型: emission / equipment / weather |

**允许的文件扩展名**: csv, xlsx, xls

**请求示例 (curl)**:

```bash
# CSV 排放数据
curl -X POST http://localhost:5000/api/data/upload \
  -F "file=@emission_data.csv" \
  -F "data_type=emission"
```

**成功响应 (200)**:

```json
{
  "message": "导入成功",
  "imported_rows": 2160,
  "before_completeness": 95.42,
  "after_completeness": 100.0
}
```

**错误响应**:

| HTTP | error | 说明 |
|:--:|------|------|
| 400 | "缺少文件" | 未上传文件 |
| 400 | "无效文件，仅支持 .csv / .xlsx / .xls" | 文件格式不支持 |
| 400 | "缺少或无效的 data_type" | data_type 值不合法 |
| 400 | "文件为空" | 文件内容为空 |
| 500 | "处理失败: ..." | 服务器内部错误 |

### 2.2 JSON 实时数据接入

```
POST /api/data/realtime
```

**请求格式**: `application/json`

**请求体结构**:

```json
{
  "data_type": "emission | equipment | weather",
  "records": [
    { "字段": "值", ... }
  ]
}
```

**必填字段 (按 data_type)**:

| data_type | 必填字段 |
|-----------|----------|
| emission | sensor_id, timestamp |
| equipment | equipment_id, timestamp |
| weather | station_id, timestamp |

**请求示例**:

```json
{
  "data_type": "emission",
  "records": [
    {
      "sensor_id": "S01",
      "timestamp": "2026-05-28 14:00:00",
      "voc_concentration": 65.3,
      "nox_concentration": 26.1,
      "so2_concentration": 4.6
    }
  ]
}
```

**成功响应 (200)**:

```json
{
  "message": "实时数据接入成功",
  "processed_records": 1
}
```

**错误响应**:

| HTTP | 说明 |
|:--:|------|
| 400 | 请求体不是 JSON |
| 400 | 缺少或无效的 data_type |
| 400 | records 必须是非空列表 |
| 400 | 第 N 条记录缺少必填字段 |
| 500 | 数据处理失败 |

### 2.3 获取最新实时数据

```
GET /api/realtime
```

**请求示例**:

```bash
curl http://localhost:5000/api/realtime
```

**成功响应 (200)**:

```json
{
  "voc": 64.4,
  "temp": 29.9,
  "humidity": 47.0,
  "wind": 2.7
}
```

**字段说明**:

| 字段 | 类型 | 单位 | 说明 |
|------|------|------|------|
| voc | Float | mg/m³ | 最新 VOCs 浓度 |
| temp | Float | °C | 最新温度 |
| humidity | Float | % | 最新相对湿度 |
| wind | Float | m/s | 最新风速 |

**错误响应**: 500 返回 `{"error": true, "message": "..."}`

---

### 2.4 预测数据 (历史+预测)

```
GET /api/prediction
```

**请求示例**:

```bash
curl http://localhost:5000/api/prediction
```

**成功响应 (200)**:

```json
{
  "history": [
    {"time": "05/28 10:00", "value": 62.3},
    {"time": "05/28 11:00", "value": 58.7},
    ...
  ],
  "prediction": [
    {"time": "05/28 15:00", "value": 55.1},
    {"time": "05/28 16:00", "value": 54.8},
    {"time": "05/28 17:00", "value": 54.2},
    {"time": "05/28 18:00", "value": 53.5},
    {"time": "05/28 19:00", "value": 52.9},
    {"time": "05/28 20:00", "value": 52.1}
  ]
}
```

**说明**:
- `history`: 最近 24 条历史浓度数据
- `prediction`: 未来 6 小时预测值
- 若模型未训练返回 history 为空，prediction 为空
- 若数据不足 24 条返回所有可用历史
- 单次响应时间 < 2 秒

### 2.5 预测 + 预警一体化

```
GET /api/vocs/prediction
```

**请求示例**:

```bash
curl http://localhost:5000/api/vocs/prediction
```

**成功响应 (200)**:

```json
{
  "error": false,
  "predictions": [
    {"hour": "+1h", "voc_concentration": 55.09},
    {"hour": "+2h", "voc_concentration": 55.17},
    {"hour": "+3h", "voc_concentration": 54.89},
    {"hour": "+4h", "voc_concentration": 54.49},
    {"hour": "+5h", "voc_concentration": 54.46},
    {"hour": "+6h", "voc_concentration": 54.36}
  ],
  "prediction_elapsed_ms": 99.62,
  "alert": {
    "triggered": false,
    "level": null,
    "exceed_count": 0,
    "records": []
  },
  "total_elapsed_ms": 115.47
}
```

**预警等级规则**:

| 超标个数 | 预警等级 |
|:--:|:--:|
| >3 | 高 |
| 2-3 | 中 |
| 1 | 低 |
| 0 | 不触发 |

**错误响应**:

| HTTP | 说明 |
|:--:|------|
| 400 | 有效数据不足 |
| 500 | 预测失败 |
| 503 | 模型文件缺失（需先训练） |

---

### 2.6 预警记录列表

```
GET /api/alerts
```

**请求示例**:

```bash
curl http://localhost:5000/api/alerts
```

**成功响应 (200)**:

```json
[
  {
    "id": 1,
    "time": "05/28 14:00",
    "metric": "VOCs浓度",
    "level": "warn",
    "detail": "预测超标时间: 05/28 15:00, 预测值: 85.2 mg/m³"
  }
]
```

**level 取值**: `error` (高), `warn` (中), `info` (低)

**排序**: 按预警时间倒序，最多返回 50 条

---

### 2.7 CSV 报告导出

```
GET /api/report/export
```

**请求示例**:

```bash
curl http://localhost:5000/api/report/export -o gas_report.csv
```

**成功响应 (200)**:

```text
Content-Type: text/csv
Content-Disposition: attachment; filename=gas_report.csv

时间,VOCs浓度(mg/m³),NOx浓度(mg/m³),SO2浓度(mg/m³)
2026-03-01 00:00:00,45.2,17.2,3.2
2026-03-01 01:00:00,42.8,16.3,3.0
...
```

**说明**: 返回最近 500 条历史排放数据，CSV 格式。

## 三、错误码规范

| HTTP 状态码 | 含义 | 触发条件 |
|:--:|------|------|
| 200 | 成功 | 请求正常处理 |
| 400 | 请求错误 | 参数缺失/格式错误/文件类型不支持 |
| 500 | 服务器内部错误 | 数据库异常/代码异常 |
| 503 | 服务不可用 | 模型文件缺失（需训练模型） |

**统一错误响应格式**:

```json
{
  "error": true,
  "message": "具体错误描述"
}
```

## 四、OpenAPI 3.0 规范 (部分)

```yaml
openapi: "3.0.0"
info:
  title: Gas System API
  description: 面向化工园区的多源废气智能治理系统 API
  version: "1.0.0"
servers:
  - url: http://localhost:5000
    description: 本地开发服务器
  - url: http://localhost
    description: Docker Nginx 代理

paths:
  /api/realtime:
    get:
      summary: 获取最新实时数据
      responses:
        "200":
          description: 成功
          content:
            application/json:
              schema:
                type: object
                properties:
                  voc:
                    type: number
                    description: VOCs浓度 (mg/m³)
                  temp:
                    type: number
                    description: 温度 (°C)
                  humidity:
                    type: number
                    description: 相对湿度 (%)
                  wind:
                    type: number
                    description: 风速 (m/s)

  /api/prediction:
    get:
      summary: 获取历史+预测数据
      responses:
        "200":
          description: 成功
          content:
            application/json:
              schema:
                type: object
                properties:
                  history:
                    type: array
                    items:
                      $ref: "#/components/schemas/TimeValue"
                  prediction:
                    type: array
                    items:
                      $ref: "#/components/schemas/TimeValue"

components:
  schemas:
    TimeValue:
      type: object
      properties:
        time:
          type: string
          example: "05/28 14:00"
        value:
          type: number
          example: 62.3
```
