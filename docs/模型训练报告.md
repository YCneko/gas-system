### 1.5 数据清洗流程

```
原始数据
  → 时间戳统一 (pd.to_datetime)
  → 缺失值填充 (前向填充 → 后向填充 → 均值兜底)
  → 异常值处理 (负值 → NaN → 线性插值)
  → 分钟级时序对齐 (dt.floor('min'))
  → 三表按分钟合并 (merge on minute)
  → 小时级降采样 (groupby hour → mean)
  → 清洗后完整性: 100%
```

## 二、模型结构设计

### 2.1 模型选型

采用 **LSTM (Long Short-Term Memory)** 递归神经网络，适合多变量时序预测任务。

### 2.2 网络架构

```
Model: Sequential
┌──────────────────────┬─────────────────┬────────────┐
│ Layer (type)         │ Output Shape    │ Param #    │
├──────────────────────┼─────────────────┼────────────┤
│ lstm_1 (LSTM)        │ (None, 24, 128) │ 73,216     │
│ dropout_1 (Dropout)  │ (None, 24, 128) │ 0          │
│ lstm_2 (LSTM)        │ (None, 64)      │ 49,408     │
│ dropout_2 (Dropout)  │ (None, 64)      │ 0          │
│ dense_1 (Dense)      │ (None, 32)      │ 2,080      │
│ output (Dense)       │ (None, 6)       │ 198        │
├──────────────────────┼─────────────────┼────────────┤
│ Total params         │                 │ 124,902    │
│ Trainable params     │                 │ 124,902    │
│ Non-trainable params │                 │ 0          │
└──────────────────────┴─────────────────┴────────────┘
```

### 2.3 设计原则

- **LSTM 层数**: 2 层，平衡表达能力与计算开销
- **Dropout 率**: 0.3，防止小数据集过拟合
- **参数量控制**: 124,902 < 500,000，满足要求
- **多层输出**: Dense(6) 直接输出 6 个未来值，简化训练

## 五、模型评估

### 5.1 测试集评估

| 指标 | 值 | 说明 |
|------|:--:|------|
| MSE (均方误差) | 258.12 | 预测值与真实值偏差的平方均值 |
| MAE (平均绝对误差) | 12.40 mg/m³ | 平均预测偏差 |
| RMSE (均方根误差) | 16.07 mg/m³ | MSE 的平方根 |
| MAPE | 17.39% | 平均绝对百分比误差 |
| **准确率 (1-MAPE)** | **82.61%** | **超过 75% 要求** |
| R² Score | -0.26 | 决定系数 (负值表示模型波动大于数据方差) |

### 5.2 需求对照

| 需求指标 | 要求值 | 实测值 | 结论 |
|------|------|------|:--:|
| 预测准确率 (1-MAPE) | ≥75% | 82.61% | ✓ 达标 |
| 单次推理时间 | ≤2 秒 | 99.62ms | ✓ 达标 |
| 模型参数量 | <50 万 | 124,902 | ✓ 达标 |
| 包含 LSTM 层数 | ≥2 | 2 | ✓ 达标 |
| 包含 Dropout | 是 | 0.3 × 2 | ✓ 达标 |

### 5.3 R² 分析

R² 为 -0.26 表明模型的方差解释能力不足。原因分析：

1. **数据量有限**: 2,160 条小时级数据对于深度学习来说偏少
2. **输出维度高**: 同时预测 6 个时间步，增大了误差
3. **VOCs 固有波动**: 即使物理模型生成的数据也包含显著随机噪声

**改进方向**:
- 增大训练数据集 (建议 ≥5,000 条)
- 使用 Sequence-to-Sequence 架构替代直接多输出
- 添加注意力机制 (Attention) 捕捉长期依赖
- 集成气象预报数据作为额外输入
