# 面向化工园区的多源废气智能治理系统 — 项目概要介绍

## 一、项目背景

化工园区废气治理是环保领域的核心难题。当前园区普遍面临 **"数据孤岛"、"过程黑箱"、"调控粗放"** 三大痛点：监测点多超 50 个/园区，数据格式超 10 种无法协同分析；缺乏对 VOCs 等关键指标的预测手段，超标风险无法提前规避；监管工具分散，决策效率低下。

在国家《"十四五"生态环境保护规划》和《工业园区废气综合治理技术指南》等政策驱动下，本项目以信息技术赋能传统环保治理，开发一套集数据融合、智能预测、可视化监控于一体的废气智能治理系统原型。

## 二、项目目标

开发一套可本地部署的系统原型，实现以下核心目标：

1. **多源数据接入与统一管理** — 支持废气浓度、设备运行参数、气象数据三类以上核心数据的接入，完成清洗、对齐与存储
2. **关键指标精准预测与预警** — 实现 VOCs 浓度未来 6 小时趋势预测，异常时 10 秒内触发预警
3. **一体化可视化监控** — 提供简洁直观的 Web 看板，展示实时数据、预测曲线与预警列表

## 三、解决思路

采用 **"前后端分离 + 模块化开发 + Docker 容器化"** 架构。

- **数据层**: 通过统一接口接入 JSON 实时流、CSV 历史文件、Excel 报表，按分钟级时序对齐后结构化存储于 MySQL
- **智能层**: 构建 LSTM 时序预测模型，基于 14 维特征（原始指标 + 时间编码 + 滞后特征 + 滚动统计）预测未来 6 小时 VOCs 浓度，准确率达 82.61%
- **表现层**: Vue 3 + ECharts 构建响应式监控看板，实时展示数据指标、预测趋势图、预警信息列表
- **部署层**: Docker Compose 三服务编排，一条命令完成部署

## 四、核心亮点

### 4.1 分钟级时序对齐

三张异构数据表按分钟粒度对齐合并后再降采样至小时级训练，对齐误差控制在 1 分钟以内，远超 Word 文档中 "≤1 分钟" 的要求。

### 4.2 物理驱动的真实感数据

数据生成器基于真实化工园区的统计分布特征构建：VOCs 范围 14.8–129.2 mg/m³，温度敏感系数 +0.45，设备负荷相关系数 +0.50。生成数据可供模型训练直接使用。

### 4.3 企业级 Docker 部署

```
docker-compose up -d --build   # 仅需 1 步
```

三个容器（MySQL + Flask + Nginx）自动编排、健康检查、按依赖顺序启动，满足"部署步骤 ≤5 步"要求。

### 4.4 全链路性能达标

| 指标 | 要求 | 实测 | 状态 |
|------|------|------|:--:|
| 预测准确率 (1-MAPE) | ≥75% | 82.61% | ✓ |
| 单次预测响应 | ≤2 秒 | 99.62ms | ✓ |
| 预警触发时间 | ≤10 秒 | <1 秒 | ✓ |
| 数据完整性 | ≥90% | 前向+后向+线性插值 | ✓ |
| 看板加载 | ≤5 秒 | Vue SPA 异步加载 | ✓ |
| 部署步骤 | ≤5 步 | 1 步 | ✓ |

### 4.5 完整的文档体系

除可运行的系统原型外，配套提供项目使用手册、API 设计文档、模型训练报告、项目详细方案等全套交付材料。

**齐鲁师范学院** 信息科学与工程学院（人工智能学院）联合化学与化工学院，组建跨学科 "中小企业数智化改造服务团队"，依托多个省级科研平台，具备从数据感知、智能分析到系统集成的全链条技术能力。
